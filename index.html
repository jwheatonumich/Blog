First Website Complete

April 28, 2020

Site Structure

I've completed the remainder of the HTML and CSS lessons on Free Code Camp. I've also finished my first project: a tribute page to Cy Twombly. Rather than keep posting my notes and projects in the blog, I'm going to create two pages linked in the nav to compile projects and notes.

I'm also going to start improving the structure and style of the blog based on what I've learned. Early steps will include making the content scale based on screen size and incorporating a CSS-gridbox to organize the various sections on the page. The color scheme could use some help too!

Making something to be proud of

I'm learning that creating even a basic website takes time. And even more time if I want it to look good. I've decided to do more than just fulfil each of the project requirements and to turn the projects into pieces that I would be proud to have in my portfolio, where possible. Even where the project probably won't turn into a portfolio piece (like the CFA Survey page) I am still going to try to create a professional and finished product that I can be proud of.

One obstacle to making portfolio level projects is that I only get my own feedback. I'm trying to remedy this by posting the projects to the Free Code Camp forums and also getting my wife's feedback. I'm also going to look for additional ways to get constructive feedback. Maybe a larger online community (Reddit?), a web development community, or someone else who is similarly trying to learn and working together? I've also thought of looking for a mentor.

Gridboxes

One of my favorite CSS tricks I've learned through the course is the gridbox. I've always thought that HTML was just structured through a series of invisible tables, but it turns out that I can create a grid on the page and attach each section of content to points on it. Right now it takes me a lot of iterations to get the grid to work the way I want, so I'll need to find ways to get more practice. I think this site is a likely first candidate.

Testing and External Tools

When I was testing my own webpage I thought it looked great on my phone and computer, but I read a comment about a different site that made me wonder how it would look on an extremely large screen. Using a site called Responsive Test Tool I realized that it looked pretty bad. I think I'll add another new section to this site for useful tools so I don't lose track of them.

One Small Step

April 23, 2020

Over a month of lockdown

This virus has had me locked in my home for so long that I'm attempting to learn to develop websites so I don't go stir-crazy.

A week ago I started the tutorials on Free Code Camp and it is a lot more fulfilling than I remember from my most recent attempt (a week of sleep-away camp in middle school.)

I was going to dig out yet another notebook to take notes, but since I'm already filling one with my attempt to study for the CFA level 1 exam this December I decided to try to take notes in HTML itself.

Attempts at notes for the first 5 sections on Responsive Web Design are here:

I've been storing the data on GitHub so I can access it on multiple computers. It's a nostalgic blast from back when I was a miserable consultant trying to learn data analysis through a Udemy course. Never finished that one... but at least I remember what a Git is! And they have this great new desktop app so I don't have to memorize a bunch of code just to properly save my file. I'm going to try to document my progress here. If I don't lose interest too quickly I hope it could be a nice way to both apply what I learn to this site and also record what it was like learning.
